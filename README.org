* Overview

** Description

The *macros* in Org allow you to add text or code to an Org document that will be
*expanded during export*: the text between the 3 pairs of accolades is expanded
so that readers can see it in the exported versions of the document.

Some people (such as me) use it to construct complex HTML or LaTeX code.

** Objectives

Using macros allows you to *insert dynamic content* or *add extra functionality* in
exports of an Org document.

** Features

These [[https://github.com/fniessen/org-macros][Org-macros]] have been tested for HTML exports.

FIXME They will be soon made available for PDF exports.

** Checkout

Use this command to check out the latest source code of the project from the
Git repository hosted at [[https://github.com/fniessen/org-macros][GitHub]]:

Load macro definitions via:

#+begin_src org :tangle no

#+end_src

#+end_info

** Usage

Once you've cloned [[https://github.com/fniessen/org-macros][Org-macros]], many macros will be ready to use within your Org
files.

You can *invoke macros* anywhere in the document (even in code examples).

Examples:

- With no arguments:

  #+begin_example
  {{{macro-name}}}
  #+end_example

- With arguments:

  #+begin_example
  {{{macro-name(arg1,arg2)}}}
  #+end_example

** Ik macro
:PROPERTIES:
:CUSTOM_ID: ik
:END:

The ~ik~ macro formats a keyboard input and simultaneously creates an index entry
when exporting to LaTeX.

It provides a unified syntax for documenting keyboard input across LaTeX and
HTML backends.

In LaTeX, it combines ~\index~ and ~\keys~.

In HTML, it renders the key sequence using the ~<kbd>~ element.

*** Code

#+begin_src org
,#+MACRO: ik  @@latex:\index{$1}\keys{$1} @@@@html:<kbd>$1</kbd>@@
#+end_src

*** Parameters

1. Key or key sequence to display and index.

*** Examples

#+begin_src org :tangle no
Press {{{ik(C-c a a)}}} to launch the agenda.
#+end_src

LaTeX export:

- Inserts the key sequence using ~\keys{C-c a a}~
- Adds an index entry ~C-c a a~

HTML export:

- Renders ~<kbd>C-c a a</kbd>~

*** See also

[[#k][k]] macro

** Sup macro
:PROPERTIES:
:CUSTOM_ID: sup
:END:

The ~sup~ macro produces a superscript while preventing unwanted ligatures or
spacing issues in LaTeX by inserting a zero-width non-joiner before the
superscript.

It is mainly intended for LaTeX-derived backends and is useful for footnote
markers, ordinal indicators, or mathematical-style annotations in running text.

*** Code

#+begin_src org
,#+MACRO: sup \zwnj^{$1}
#+end_src

*** Parameters

1. Text to be rendered as superscript.

*** Examples

#+begin_src org :tangle no
E = mc{{{sup(2)}}}
#+end_src

LaTeX export:

#+begin_src latex :tangle no
E = mc\zwnj^{2}
#+end_src

The zero-width non-joiner prevents TeX from incorrectly joining the superscript
with the preceding glyph.

** If-latex-else macro
:PROPERTIES:
:CUSTOM_ID: if-latex-else
:END:

The ~if-latex-else~ macro conditionally expands its content depending on whether
the current export backend is derived from LaTeX.

It allows writing backend-specific content inline, without duplicating sections
or using export blocks.

*** Code

#+begin_src org
,#+MACRO: if-latex-else (eval (if (org-export-derived-backend-p org-export-current-backend 'latex) "$1" "$2"))
#+end_src

*** Parameters

1. Code expanded when exporting to a LaTeX-derived backend (LaTeX, Beamer, etc.).
2. Code expanded when exporting to any other backend.

*** Examples

#+begin_src org :tangle no
{{{if-latex-else(latex code,something else)}}}
#+end_src

When exporting to LaTeX or Beamer, this expands to:

#+begin_src latex :tangle no
latex code
#+end_src

When exporting to another backend, it expands to:

#+begin_src text :tangle no
something else
#+end_src

** P (Beamer overlay) macro
:PROPERTIES:
:CUSTOM_ID: P
:END:

The ~P~ macro provides a shortcut for inserting an inline Beamer pause in order to
create successive overlays in slides exported with ox-beamer.

It wraps the Beamer ~\pause~ command using Org's backend-specific inline syntax.

The macro is intended to simplify the writing of slides where sentences appear
progressively, without using lists, blocks, or dedicated Beamer environments.

*** Code

#+begin_src org
,#+MACRO: P @@beamer:\pause@@
#+end_src

*** Parameters

This macro takes no parameters.

It performs a direct textual substitution into Beamer inline code at export time.

*** Examples

#+begin_src org :tangle no
First sentence. {{{P}}}
Second sentence. {{{P}}}
Third sentence.
#+end_src

When exporting to Beamer, each occurrence of ~{{{P}}}~ is replaced with \pause,
causing the sentences to appear on successive overlays.

:PROPERTIES:

The ~style~ macro allows importing an external CSS style sheet into an HTML

#+begin_src org

#+end_src

*** Parameters

*** Examples

#+begin_src org :tangle no

#+end_src

#+begin_src org :tangle no
,#+HTML_HEAD_EXTRA: <link type="text/css" rel="stylesheet" href="http://whatever.com/mystyle.css" media="all">
#+end_src

The referenced style sheet is then loaded by the generated HTML page.

** Vspace macro
:PROPERTIES:
:CUSTOM_ID: vspace
:END:

The ~vspace~ macro adds a fixed vertical gap to your Org document when exporting.

*** Code

#+begin_src org
,#+MACRO: vspace @@latex:\vspace{10pt}@@@@html:<div style="margin-bottom: 10pt;"></div>@@
#+end_src

*** Parameters

None. The 10 pt value is baked into the macro definition.

*** Examples

Insert space between paragraphs:

#+begin_src org :tangle no
First paragraph.

{{{vspace}}}

Second paragraph.
#+end_src

*** Customization

To change the gap, redefine the macro in your setup:

#+begin_src org
,#+MACRO: vspace @@latex:\vspace{20pt}@@@@html:<div style="margin-bottom: 20pt;"></div>@@
#+end_src

** Vsp macro
:PROPERTIES:
:CUSTOM_ID: vsp
:END:

The ~vsp~ macro inserts vertical space in Org-mode exports (LaTeX and HTML). It
takes one optional parameter -- the amount of space to insert.

*** Code

#+begin_src org
,#+MACRO: vsp @@latex:\vsp{$1}@@@@html:<div style="margin-bottom: $1;"></div>@@
#+end_src

*** Parameters

~$1~: any valid TeX/CSS length (e.g. ~10pt~, ~1em~, ~2cm~, ~15px~).

*** Examples

Insert space between paragraphs:

#+begin_src org :tangle no
First paragraph.

{{{vsp(20pt)}}}

Second paragraph appears after 20 pt of extra space.

{{{vsp(1.5em)}}}
#+end_src
